# AI Prompts Used

## Prompt One: Initial Application

Work only inside the week3-task-manager folder.

First inspect the existing React and Vite files. Briefly explain your implementation plan before changing anything.

Build a responsive Study Task Manager application using React.

The application should allow users to add tasks, select priorities, mark tasks as completed, delete tasks, filter tasks, view task statistics, and preserve tasks using localStorage. It should include validation, accessible form controls, an empty state, and a responsive design.

After implementing the feature, run the production build, fix any errors, explain the files changed, and report the final build result.

## Prompt Two: Review and Correction

Review the Study Task Manager implementation inside the week3-task-manager folder.

Check for React state management mistakes, localStorage errors, accessibility issues, whitespace-only task validation, incorrect task counts, filter problems, and mobile layout problems.

Do not redesign the application unnecessarily. Make only corrections that improve reliability, accessibility, or usability. Run the production build and report every correction made.
