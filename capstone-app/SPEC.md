# StudyFlow Specification

| Screen name | Route | Current purpose | Future functionality |
| --- | --- | --- | --- |
| Landing page | `/` | Introduces StudyFlow and links to core student productivity areas. | Personalized onboarding, recent activity, and quick-start actions. |
| Dashboard overview | `/dashboard` | Shows a placeholder summary for progress, deadlines, and study priorities. | Live study metrics, deadline alerts, streaks, and cross-feature summaries. |
| Study tasks | `/tasks` | Reserves space for study tasks, assignments, readings, and revision work. | Task creation, editing, completion, filtering, priorities, and due dates. |
| Study calendar | `/calendar` | Reserves space for classes, exams, study blocks, and academic dates. | Month, week, and agenda views with linked tasks and reminders. |
| Account and notification settings | `/settings` | Reserves space for profile and notification preferences. | Account management, notification schedules, reminder channels, and accessibility preferences. |
| Application health status | `/health` | Displays a placeholder section and browser-fetched API health status. | Expanded diagnostics, uptime history, and dependency checks. |
| Health API | `/api/health` | Returns JSON with status, application name, timestamp, and environment. | Add dependency health, version metadata, and deployment identifiers. |
